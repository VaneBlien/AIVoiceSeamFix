

现在的修复流程是：

```text
断裂检测 → 构建区域 → 高斯局部平滑 → 与原音频按 alpha 混合
```

如果修复后声音变得“清脆”，常见原因有三个：

第一，断裂附近的低频拖尾或浑浊瞬态被平滑掉了。拼接点本来可能有很短的爆点、相位跳变、低频突刺，处理掉之后，人耳会觉得声音更干净、更亮。

第二，当前只处理第一声道并重新输出 mono。如果原文件是 stereo，而两个声道有轻微相位差，转成 mono 或只取左声道后，可能听起来更集中、更靠前。

第三，`window_ms=60`、`sigma_ms=4`、`alpha=0.45` 这个组合偏“有效修复”，不是特别保守。它会让断点附近的毛刺和局部能量变化被压平，听感可能从“糊/裂/杂”变成“干净/脆”。

我建议后面给 GUI 加两个预设：

```text
轻度修复：
sensitivity = 10.0
expand_ms   = 30.0
merge_ms    = 60.0
window_ms   = 35.0
sigma_ms    = 2.5
alpha       = 0.25
```

```text
标准修复：
sensitivity = 8.0
expand_ms   = 50.0
merge_ms    = 80.0
window_ms   = 60.0
sigma_ms    = 4.0
alpha       = 0.45
```

如果你觉得“清脆”是好听的，那可以保留现在的标准参数；如果觉得有点过度处理，就先把 `alpha` 降到 `0.25~0.35`，再把 `sigma_ms` 降到 `2.0~3.0`。这两个参数最影响“修复味”。
